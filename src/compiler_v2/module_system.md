# L2 模块系统设计

## 1. 模块系统概述

L2 模块系统参考 Rust 的模块系统设计，支持多文件编译和链接。

## 2. 模块语法

### 2.1 模块声明

```l2
// 单文件模块声明
模块 utils {
    函数 加(a: 整数, b: 整数) -> 整数 {
        返回 a + b
    }
}

// 多文件模块声明
模块 math
```

### 2.2 模块导入

```l2
// 导入整个模块
使用 utils

// 导入特定函数
使用 utils::加

// 导入并改名
使用 utils::加 作为 加法
```

## 3. 多文件组织结构

### 3.1 目录结构

```
project/
├── main.xy         # 主入口文件
├── utils.xy        # 单文件模块
└── math/           # 多文件模块目录
    ├── mod.xy      # 模块入口
    ├── add.xy      # 子模块
    └── sub.xy      # 子模块
```

### 3.2 模块解析规则

1. **单文件模块**：直接在文件中定义模块内容
2. **多文件模块**：
   - 目录名即为模块名
   - 目录中必须包含 `mod.xy` 文件作为模块入口
   - 目录中的其他 `.xy` 文件自动成为子模块

## 4. 编译流程

### 4.1 单文件编译

1. 词法分析 → 语法分析 → 语义分析 → 代码生成 → 链接

### 4.2 多文件编译

1. **模块解析**：解析所有模块声明和导入
2. **依赖分析**：构建模块依赖图
3. **并行编译**：按依赖顺序编译各个模块
4. **IR 合并**：将所有模块的 IR 合并
5. **链接**：生成最终可执行文件

## 5. 符号解析

### 5.1 模块级符号

- 每个模块拥有独立的命名空间
- 符号访问方式：`模块名::符号名`
- 支持多级模块：`a::b::c`

### 5.2 外部函数声明

```l2
外部 函数 printf(format: 文本, ...) -> 整数
```

## 6. 实现计划

1. **词法分析器扩展**：添加 `模块`、`使用` 关键字
2. **语法分析器扩展**：支持模块声明和导入语法
3. **模块解析器**：实现模块文件查找和解析
4. **依赖分析器**：构建模块依赖图
5. **代码生成器修改**：支持模块级符号
6. **链接器修改**：支持多文件 IR 合并
7. **测试验证**：验证多文件编译和链接功能

## 7. 示例

### 7.1 多文件示例

**main.xy**:
```l2
使用 math
使用 utils

函数 主(): 整数 {
    打印(utils::加(1, 2))
    打印(math::add(3, 4))
    返回 0
}
```

**utils.xy**:
```l2
模块 utils {
    函数 加(a: 整数, b: 整数) -> 整数 {
        返回 a + b
    }
}
```

**math/mod.xy**:
```l2
模块 math {
    使用 self::add
}
```

**math/add.xy**:
```l2
模块 math::add {
    函数 add(a: 整数, b: 整数) -> 整数 {
        返回 a + b
    }
}
```
