# XY 语言 （v0.1）规范

## 一、设计理念与核心目标

### 1.1 设计哲学

XY 语言 v0.1 版本以**自举优先**和**中文原生**为核心，兼顾实现简洁性与跨平台能力，具体原则如下：

- **自举优先**：所有特性设计以“能用 XY 写出 XY 编译器”为最高优先级，牺牲完整泛型、模式匹配等高级特性，确保实现的确定性和简洁性。

- **中文原生**：标识符、关键字、错误信息全面支持中文，消除非英语母语者的认知摩擦。

- **简化内存管理**：不引入复杂垃圾回收（GC），采用“栈分配 + 确定性析构”模型，保证资源管理可预测。

- **LLVM 亲和**：底层直接映射到 LLVM IR，复用其优化能力与跨平台特性。

### 1.2 核心术语表

| 术语       | 定义 / 说明                                                               |
| ---------- | ------------------------------------------------------------------------- |
| 文本       | 对应 UTF-8 字符串（String），规范统一使用 “文本”，避免 “字符串” 混用。    |
| 空         | 无返回值类型，等价于其他语言的 ()。                                       |
| 指针       | 不透明内存地址类型（void\*），仅用于 FFI 和内部实现，用户代码不可解引用。 |
| 列表       | 动态数组容器，v0.1 基于类型擦除实现，底层存储指针，用户需自行转换类型。   |
| 运行时库   | C 语言编写的基础库（rt.c），提供内存分配、文本操作等底层功能。            |
| 确定性析构 | 编译器在变量离开作用域时，自动插入资源释放代码。                          |
| 自举子集   | 语言的最小子集，足以编写编译器本身。                                      |

## 二、语言核心规范

### 2.1 词法与语法

#### 2.1.1 标识符

允许任意非空白、非运算符的 Unicode 字符序列。

#### 2.1.2 关键字（严禁用作标识符）

| 分类                  | 关键字列表                                                           |
| --------------------- | -------------------------------------------------------------------- |
| 结构控制              | 函数、结构体、枚举、外部、导入、若、否则、循环、当、返回、中断、继续 |
| 类型相关              | 整数、浮点、布尔、文本、空、列表、指针                               |
| 预留（v0.1 暂不支持） | 让、可变、自（使用时报错提示 “预留特性”）                            |

#### 2.1.3 字面量

- 布尔值：真、假（保留字面量，非关键字）；

- 整数：123、-45 等；

- 文本："内容"（UTF-8 编码）。

#### 2.1.4 注释

支持单行注释 `//` 和多行注释 `/* */`，完全兼容 UTF-8 字符。

### 2.2 类型系统

#### 2.2.1 基础类型

整数（i64）、浮点（f64）、布尔、文本、空、指针。

#### 2.2.2 复合类型

| 类型   | 定义与使用规则                                                                                                                                    |
| ------ | ------------------------------------------------------------------------------------------------------------------------------------------------- |
| 结构体 | 语法：结构体 名称 {字段：类型，...}；默认按值传递。                                                                                               |
| 枚举   | 语法：枚举 名称 {变体 1 (类型 1, 类型 2...)，变体 2，... }；变体支持元组风格多字段。                                                              |
| 列表   | 1. 实现：类型擦除，底层存储指针；<br>2. 使用：存入时需转为指针，取出时转回具体类型；<br>3. 推荐：统一列表 API，内部存指针（初期可提供特化版本）。 |

#### 2.2.3 传递语义

- 默认按值传递；

- 后端优化：编译器可通过 LLVM 的 byval 属性自动优化大结构体传递方式，语言层面不暴露引用符号（&）。

### 2.3 内存模型

- **栈分配**：局部变量默认通过 alloca 分配到栈上；

- **堆管理与确定性析构**：

  1. 文本、列表的内部数据分配在堆上；

  2. 编译器责任：进行作用域分析，变量离开作用域（函数返回、块结束）时自动生成资源释放代码（如 `rt_string_free(s)`）；

  3. 限制：避免循环引用（无 GC，会导致内存泄漏）。

### 2.4 内置函数与运行时 API

编译器隐式链接运行时库，用户可直接调用以下高层封装函数：

| 函数名   | 类型签名                          | 说明                          |
| -------- | --------------------------------- | ----------------------------- |
| 打印     | (内容：文本) -> 空                | 输出文本到控制台              |
| 新建列表 | () -> 列表                        | 创建空列表                    |
| 列表追加 | (list: 列表，item: 指针) -> 空    | 向列表添加元素（item 为指针） |
| 列表获取 | (list: 列表，index: 整数) -> 指针 | 获取列表指定索引元素          |
| 读取行   | () -> 文本                        | 读取控制台输入的一行文本      |
| 文本长度 | (t: 文本) -> 整数                 | 返回文本的 UTF-8 长度         |

> 注：类型转换（如整数转指针）可通过 FFI、内置转换函数或 LLVM IR 层 bitcast 实现。

## 三、自举子集规范

### 3.1 必须支持的最小特性集（S_min）

V1（Rust 实现的编译器）必须实现，V2（XY 自举编译器）仅限使用以下特性：

1. 控制流：若/否则、循环、返回、中断、继续；

2. 数据定义：结构体（多字段）、枚举（多字段变体，必需）、常量；

3. 模块化：导入 "相对路径"（类似 C #include，无命名空间隔离，路径相对于当前文件目录）；

4. FFI：支持声明并调用任意 C 函数（如 malloc、printf），支持指针类型；

5. 递归：支持函数递归调用；

6. 类型：整数、布尔、文本、指针、空、列表（指针版）。

### 3.2 自举合规性测试

#### 3.2.1 构建流程

1. `xyc_rust --input src/compiler_v2.xy --output xyc_v2`（V1 编译 V2 源码）；

2. `./xyc_v2 --input tests/hello.xy --output hello`（V2 编译测试程序）；

3. 运行 `./hello` 验证输出。

#### 3.2.2 自动化检查

CI 脚本扫描 V2 源码，通过正则匹配禁止使用未实现特性的关键字（如泛型、特质），发现则报错。

## 四、代码生成规范

### 4.1 标识符映射规则

仅针对用户定义的标识符，编译器生成的临时变量（如 %tmp0）保持 ASCII：

1. 若标识符匹配 `^[a-zA-Z_][a-zA-Z0-9_]*`，直接输出（如 @foo）；

2. 否则包裹双引号并转义特殊字符，格式为 @"名称" 或 %"名称"（无空格）。

> 示例：函数 计算 () → define void @"计算"()；变量 my-var → %"my-var" = alloca i64。

### 4.2 程序入口点

1. 优先级：优先查找“主”函数；若不存在，查找 main（需用户显式指定链接选项才作为入口）；两者皆无则报错；

2. 包装逻辑：

   - 生成全局 @main；

   - 若“主”返回空：call @"主"(); ret i32 0；

   - 若“主”返回整数：ret call @"主"()；

   - 若“主”返回其他类型：编译错误；

3. 一致性检查：编译器需验证“主”函数内所有返回语句的类型与函数签名一致。

### 4.3 调试信息（DWARF）

- Name 字段始终保留原始 UTF-8 中文名；

- 稳定模式（--stable-symbols）：linkageName 转为哈希值（如 \_cn_a1b2），Name 仍为中文名，IR 引用使用哈希名。

> 示例：!0 = !DISubprogram(name: "计算面积", linkageName: @"计算面积", ...)；稳定模式下 linkageName 为 @\_cn_a1b2。

## 五、FFI 与扩展规范

### 5.1 外部函数声明

- 语法：外部 函数 名称 (参数列表) -> 返回类型 [链接名]；

- 类型限制：参数/返回值仅支持整数、浮点、布尔、指针、文本（映射为 i8\*）；v0.1 不建议直接按值传递复杂结构体给 C，建议传指针；

- 链接名规则：名称含非 ASCII 时必须显式指定链接名；省略时若名称为 ASCII，默认同名。

### 5.2 AI 占位符

- 语法：#[ai("描述")]；

- 行为：函数体为空时，编译器生成调用 abort() 或抛出“Unimplemented AI Feature”的代码，并给出警告（严禁静默忽略，以防运行时错误难以排查）。

## 六、工具链与构建规范

### 6.1 项目结构与运行时库

#### 6.1.1 项目结构

```

├── src/

│   ├── main.xy       # 程序入口

│   └── lib/          # 模块文件目录

│       └── *.xy

└── runtime/

    ├── rt.c          # 运行时源码

    └── rt.h          # 运行时头文件

```

#### 6.1.2 运行时库最小接口（C 语言原型）

```c

// 字符串管理

void* rt_string_new(const char* utf8_content);  // 创建文本

void rt_string_free(void* s);                   // 释放文本

int rt_string_len(void* s);                     // 获取文本长度

// 列表管理（类型擦除，存储 void*）

void* rt_list_new();                            // 创建列表

void rt_list_append(void* list, void* item);    // 追加元素

void* rt_list_get(void* list, int index);       // 获取元素

void rt_list_free(void* list);                  // 释放列表

// IO 操作

void rt_println(void* s);                       // 打印文本

void* rt_readline();                            // 读取一行输入（返回 rt_string_new 结果）

```

### 6.2 编译流程

1. 前端：xyc 解析 .xy 文件 → 生成 LLVM IR（.ll）；

2. 汇编：通过 llc 或 LLVM API 将 IR 转为目标文件（.o）；

3. 运行时编译：`gcc -c runtime/rt.c -o rt.o`；

4. 链接：`clang main.o rt.o -lm -o app`（链接数学库等依赖）。

### 6.3 错误处理

- 格式：[错误] 文件：行：列 - 描述；

- 要求：全程中文，定位精确；

- 示例：[错误] parser.xy:105:12 - 语法错误：枚举变体 '加法' 期望字段列表，但发现 '}'。

## 七、版本演进规划

| 版本 | 阶段         | 关键交付                                              | 备注                              |
| ---- | ------------ | ----------------------------------------------------- | --------------------------------- |
| v0.1 | Foundation   | Rust 实现编译器，支持多字段枚举、指针列表、确定性析构 | 当前版本，可编译自举子集          |
| v0.2 | Bootstrap    | XY 自举编译器，引入真实泛型（列表 [T]），移除类型擦除 | 解决 v0.1 列表类型转换痛点        |
| v0.3 | Intelligence | MLIR 后端（可选切换），#[ai] 注解实际生效             | 双后端架构（LLVM + MLIR）         |
| v1.0 | Intent       | 自然语言注释驱动代码生成                              | AI 深度集成，实现 “意图驱动” 开发 |

### 总结

1. XY 语言 v0.1 核心是“自举优先 + 中文原生”，简化内存管理和高级特性，优先保证编译器自举能力；

2. 核心设计围绕 LLVM 展开，内存采用栈分配+确定性析构，列表/文本堆分配且自动释放，避免循环引用；

3. 工具链依赖 C 编写的运行时库，编译流程分为前端生成 IR、汇编、链接运行时三步，错误信息全程中文且定位精确。
